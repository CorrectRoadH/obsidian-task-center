# 用户故事

> 只讲**谁想要什么、为什么**。想知道界面长啥样 / 交互细节 → 去 [UX.md](./UX.md);想知道数据结构 / 代码怎么组织 → 去 [ARCHITECTURE.md](./ARCHITECTURE.md)。

## 我们在做什么

在 Obsidian 的 `- [ ] 任务` 这个最朴素的写法之上,加两样东西:

1. 一个**看板**,用拖拽就能排一整周、把不想做的扔进垃圾桶、回头看估时准不准。
2. 一套**命令行动词**,让 Claude Code 这类 AI agent 能稳定地读写任务,不用解析人眼看的花哨输出。

数据永远是那行 markdown。不造新格式、不建数据库,Dataview 和 Obsidian Tasks 插件照常用。

## 我们在为谁做

- **P1 · 常驻 Obsidian 的人**
  已经在用 Tasks 插件,想在 markdown 上加一层看板 + 拖拽排期,不愿切出当前工作流。
- **P2 · AI agent**
  Claude Code 或类似 agent,代替用户从 shell 读写任务。对"输出能被 grep"、"重复跑不炸"、"错误能解释"这些事很敏感。
- **P3 · 做精力管理的人**
  用 Covey 四象限,用"估了多久 vs. 真花了多久"对抗自己的乐观。关心"放弃"这个动作,不想只能"完成或删除"。

## P1 · 常驻 Obsidian 的人想要的

每条带一个稳定编号,方便 PR / commit 引用。已上线的标 ✅,已合入未发版标 🟡,还没做的标 ⬜。

### 看板 & 视图

- `US-101` ✅ 一眼看到**本周 7 天**谁塞满了,今日高亮。
- `US-102` ✅ 月度日历,方便往前看两三周。
- `US-103` ✅ 同一天内,deadline 近的自动排在前面。
- `US-104` ✅ 未排期池顶永远是"该挑下一件的那件"——先按 deadline,再按新加入时间。
- `US-105` ✅ 每个 tab 自带"待办数"小圆点,不用切过去看。
- `US-106` ✅ 状态栏一直显示 `📋 N today · ⚠ M overdue`,点一下就打开看板。
- `US-107` ✅ callout 里的 `> - [ ]` 也算数,不被忽略。

### 拖拽

- `US-121` ✅ 把卡从"周三"拖到"周五",这张卡的 `⏳` 就改成周五——不用改 markdown。
- `US-122` ✅ 月视图每个日期格都可以当拖入目标。
- `US-123` ✅ 把卡拖进底部固定的**垃圾桶**,标成"放弃"(`[-] ❌`),不是从磁盘删。
- `US-124` ✅ 放弃一个父任务时,还没打钩的子任务跟着放弃,**已经打钩的子任务不动**(作为历史保留)。
- `US-125` 🟡 把一张卡拖到**另一张卡**上,它就变成那张的子任务——两张卡在不同文件也行。
- `US-126` 🟡 不能把 A 拖到 A 自己或 A 的后代上(不允许绕圈)。
- `US-127` 🟡 每次"卡片从当前位置消失"都应该是**淡出 + 邻居平滑上移**,不该是闪一下就没了。原地不动的操作不放动画。
- `US-128` ✅ `Ctrl/Cmd+Z` 能撤销最近 20 步拖拽 / 改期 / 重命名,手滑不至于翻 git。
- `US-129` ⬜ 跨文件的子任务嵌套**不可撤销**——UI 要一次性提示我,不要悄悄做掉。

### 父子任务

- `US-141` 🟡 卡片上有个 `+ 子任务` 小按钮,点一下就能加子任务,不用记快捷键。新子任务自动**继承父的 ⏳**。
- `US-142` ✅ 子任务在卡里**递归显示**所有层级,不止一层;每层的 checkbox 和标题都能用。
- `US-143` ✅ 父可见时,子任务**不**作为独立顶层卡重复出现(嵌在父里就够了)。
- `US-144` ✅ 子的 `⏳` 和父一样 → 不显示;不一样 → 显个小 badge。
- `US-145` ✅ **完成或放弃父级,等于自动完成 / 放弃整条分支**——不用一条条勾子任务。活动视图里直接消失,已完成 tab 里能看到历史。
- `US-146` ✅ 和父级同一天建的子任务,不重复打 `➕ 创建日期` 戳,避免满屏冗余时间。
- `US-147` ✅ 改父级的时候,子任务的 tag / 估时 / 实际耗时 / emoji 字节不变。
- `US-148` → 见 US-902(待决策)。

### 编辑卡片 & 快捷键

- `US-161` ✅ **点标题就改名**,Enter 提交、Esc 回退;emoji / tag / `[estimate::]` / `[actual::]` / block 锚点 / 优先级符号一个不丢。
- `US-162` ✅ 常用动作都有键盘快捷方式:`1–4` 象限 / `←/→` 改天 / `Space` 完成 / `D` 弹日期 / `E` 打开源文件 / `Delete` 放弃 / `Ctrl+Z` 撤销 / `/` 聚焦筛选 / `Ctrl+1–4` 切 tab / `Ctrl+T` 快速添加 / `Ctrl+Shift+T` 打开看板。
- `US-163` ✅ **一行就能添加任务**,支持"今天 / tomorrow / 周六 / Mon"这种自然语言日期。

## P2 · AI agent 想要的

- `US-201` ✅ 用 Obsidian 原生的 CLI 动词,**不用 shell 套皮 / eval**。
- `US-202` ✅ list 的每一行**第一列都是稳定 id**(`path:L42` 或 hash),`grep/awk/cut` 能直接用。
- `US-203` ✅ 写操作**重复跑结果相同**。对已完成任务再跑 `done` → `ok … unchanged`,不报错。
- `US-204` ✅ 每次写都返回 `before / after` 两行 diff,方便我校验刚才到底改了啥。
- `US-205` ✅ `list` 和 `stats` 有 `format=json` 出口,我不用刮人类友好输出。
- `US-206` ✅ `stats days=N` 告诉我"估了多久 vs. 真花了多久"的比率和 top tag 分钟数,让我帮用户校准未来的估时。
- `US-207` ✅ 时间筛选词和人说话一致:`today / tomorrow / week / next-week / month / YYYY-MM-DD / FROM..TO / unscheduled`。
- `US-208` ✅ 行号失效时能**按标题 hash 找回任务**;hash 撞了给候选列表,**不要猜**。
- `US-209` ✅ 增量改时间:`actual minutes=+30m`,不用先读再写。
- `US-210` ✅ 文档里直接给出"典型一天收尾 / 快速捕捉 / 补记完成"三种工作流,我不用反推调用顺序。
- `US-211` ✅ 报错两行:`error <code>` + 一句人话;code 是短而固定的集合。
- `US-228` 🟡 有 `nest ref=A under=B` 动词,能跨文件把 A 变成 B 的子任务(GUI 拖拽的同胞)。

## P3 · 做精力管理的人想要的

- `US-301` ✅ 给任务打 `#1象限`–`#4象限`,未排期池按象限分组。
- `US-302` ✅ 记**估时** `[estimate:: 90m]` 和**实际耗时** `[actual:: 75m]`。
- `US-303` ✅ 已完成 tab 顶上看到**近 7 天的准确率**(真用 / 估),按周聚合,附 top 4 tag 分钟数。
- `US-304` ✅ 历史周默认折叠,别让过去的记录把本周挤出屏幕。
- `US-305` ✅ `[-] ❌` 是"**放弃**",和"完成"分开——回头复盘才知道自己**放弃过什么**,不是混在完成计数里。

## 跨角色的共同期望

- `US-401` ✅ **只用 markdown**,没自定义格式、没数据库。Obsidian Tasks / Dataview / 任何读纯文本的工具都还能读。
- `US-402` ✅ 中英**自动切换**(跟 Obsidian 语言),不用手动配。
- `US-403` ✅ 写操作是**原子的**,写一半崩了不会把文件写坏。
- `US-404` ✅ 读操作**跳过没有任务的文件**,大 vault 开看板不卡。
- `US-405` ✅ 关看板时记住当前 tab,**下次开在同一个**。

## 还没决定的故事

方向没定的放这里。每条给"现在是怎样、候选、谁拍板"。决定后挪回正文并分配正式编号。

### `US-901` · 循环任务(`🔁`)完成后怎么生下一个?
**现状**:解析器留着 `🔁` 字段,完成时什么都不做。
**候选**:
- A 完成时按 `🔁 every week` 在同文件追一条新 `- [ ]`,`⏳` = 下个周期。
- B 维持现状,交给 Obsidian Tasks 插件处理。
- C 设置里开关,默认 B。
**谁来拍板**:产品。

### `US-902` · 子任务拖到别的天,应该怎样?
**现状**:原地改子任务的 `⏳`。
**候选**:
- A(当前)改子任务的 `⏳`,结构不变。
- B 目标日新建一张 `[[父标题]] > 子标题` wikilink 卡,原子任务 `⏳` 不动。
- C 按修饰键区分(Alt 拖 = B)。
**谁来拍板**:产品 + 交互。

### `US-903` · 优先级符号(`🔺⏫🔼🔽⏬`)要不要真用?
**现状**:保留字符,但排序 / 筛选都没接。
**候选**:
- A 排序增加"优先级 > deadline > 创建时间"三级。
- B 加 `↑/↓` 快捷键改优先级。
**谁来拍板**:产品。

### `US-904` · 设置改完要不要不关 / 开也能生效?
**现状**:改完要关看板再开。
**候选**:看板订阅设置变更,局部重渲染。
**工作量**:中(需要把 view 拆成 controller + renderer)。
**谁来拍板**:工程。

### `US-929` · 跨文件嵌套能不能撤销?
**现状**:撤销栈不覆盖这种操作,提示"用 git"。
**候选**:
- A session 内记录 `(srcFile, srcLine, dstFile, dstLine)` 四元组,`Ctrl+Z` 逆向 nest。
- B 维持现状,只做 US-129 的提示。
**谁来拍板**:工程。

### `US-930` · 多设备同步冲突怎么办?
**现状**:Obsidian Sync / iCloud / git 三路抢同一行时行为不稳定。
**候选**:
- A 写前校验 mtime,冲突 → `error file_modified`,让用户 / agent 决定重试。
- B 维持"最后写赢",文档化。
**谁来拍板**:产品 + 工程。

### `US-940` · 上架官方 Community Plugins 要不要改名?
**现状**:plugin id 含 `obsidian` 子串,被官方目录拒收。
**候选**:改名解锁上架,会打破现有用户的插件目录路径,需要迁移脚本 / 迁移文档。
**谁来拍板**:产品。

## 故事变更怎么走

1. 新故事:取下一个空编号,写进对应章节,默认 ⬜。
2. 改验收预期:保留编号,改描述,底下加一行"🔧 改过:YYYY-MM-DD 原因 ...";避免偷偷换意思。
3. 作废:改成 `[DROPPED]` 保留条目,加一行"🗑 作废:YYYY-MM-DD 原因 ..."。
4. 从"还没决定"转正:分配正式段编号,原条目改"→ 见 US-XXX"。
5. PR / commit 用 `US-XXX:` 开头,双向追溯。
