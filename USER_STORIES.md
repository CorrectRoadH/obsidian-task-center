# 用户故事

从已发布的产品(README + CHANGELOG)倒推出来的,不是自上而下的 spec。用作优先级判断和回归测试的参照镜。

## 用户画像

- **P1 · 知识工作者** — 常驻 Obsidian,已经在用 Tasks 插件,希望在 markdown 之上加一层看板 + 拖拽排期,不想离开当前工作流。
- **P2 · AI 协作者** — Claude Code 或同类 agent。代替用户从 shell 读写任务。在意稳定 id、幂等性、grep 友好输出。
- **P3 · 精力管理用户** — 用 Covey 四象限规划,用"估时 vs 实际耗时"对抗 planning fallacy,把"放弃"视为一等状态(不是删除)。

## P1 · 知识工作者

### 看板视图

- 作为用户,我想要**本周看板(Mon–Sun、今日高亮)**,一眼看出哪天被塞满。→ `本周 / Week` tab
- 作为用户,我想要**月度日历看板**,方便往前规划更远。→ `本月 / Month` tab
- 作为用户,我希望**每一天内部的卡片按 deadline 排序**,当天最紧急的项始终在列首。
- 作为用户,我希望**未排期池按紧急度排序**(deadline 优先,其次按创建时间倒序),这样池顶永远是我该挑的下一件事。
- 作为用户,我希望**每个 tab 标题上显示待办数量 badge**,不切换 tab 也能知道工作量分布在哪。
- 作为用户,我希望**状态栏显示 `📋 N today · ⚠ M overdue`**,点击即打开看板。
- 作为用户,我希望 **callout 里的任务(`> - [ ]`)也是一等公民**,不要被忽略。

### 拖拽

- 作为用户,我想**把卡片在本周列之间拖**来改排期,不用去编辑 markdown。→ 直接改写源行的 `⏳`
- 作为用户,我希望**月视图的每个日期格都是 drop zone**,这样同一个手势能规划更远。
- 作为用户,我想**把卡片拖进底部固定的垃圾桶**标记 `[-] ❌ today`——是"放弃",不是从磁盘删除。
- 作为用户,我希望**拖入垃圾桶时级联处理未完成子任务**,这样一次手势就能退役整个项目;但**已完成(`[x]`)的子任务必须保留**作为历史。
- 作为用户,我想**把一张卡拖到另一张卡上,让它成为子任务**——跨文件也要支持,源行原子性地从原文件删除、插入到目标文件。
- 作为用户,我希望**拒绝循环和自我嵌套**,不能把 A 嵌套到 A 自己或它的后代下面。
- 作为用户,我希望**所有卡片移除动作(拖去垃圾桶、完成、放弃、拖到别的天、`←/→`、`Space`、`Delete`、日期弹窗)都淡出 + 折叠**(Web Animations API),让邻居平滑上移。原地不动的操作跳过动画。
- 作为用户,我希望 **`Ctrl/Cmd+Z` 能撤销最近 20 步拖拽 / 方向键改期 / 重命名**,手滑一下不至于要翻 git。
- 作为用户,我**知道"跨文件 nest"不可撤销**(CHANGELOG 已标注,只能用 git 或 Obsidian 文件历史)。希望这个限制在 UI 里提示我,而不是悄悄吞掉。

### 多级任务(父 / 子)

- 作为用户,我想**通过卡片上的 `+ 子任务` 按钮直接加子任务**,不用记快捷键。新子任务继承父级的 `⏳`。
- 作为用户,我希望**嵌套子任务在卡片里递归渲染**(不是只渲染一层),每一层的 checkbox 和标题编辑都要能用。
- 作为用户,我希望**父级可见时,它的子任务在视图里不再作为独立顶层卡片出现**(在父卡内部渲染即可)。
- 作为用户,我希望**子任务的 `⏳` badge 只在和父级不同时才显示**——相同时保持安静,不同时才显眼。
- 作为用户,我希望**祖先终态传播** —— 任意祖先(任务、bullet、`#dropped` 章节标题)一旦是 `[x]` / `[-]` / `#dropped`,它所有后代自动从活动视图(todo / week / month / unscheduled)消失。完成或放弃一个项目不应该还要我一条条勾子任务。
- 作为用户,我希望**子任务如果和父级是同一天创建,就不重复打 `➕` 时间戳**,避免新项目里满屏冗余时间。
- 作为用户,我希望**父级的 重命名 / 改期 / 拖拽 操作不会丢子任务的元数据**。
- 作为用户(悬而未决),我想知道**把子任务拖到另一天时到底应该发生什么**。目前是原地改写子任务的 `⏳`,而不是在目标日创建一个 `[[parent]] > child` 链接。两种都是合法的故事——我们还没决定要哪个。

### 卡片编辑与快捷键

- 作为用户,我想**点击标题就地重命名**(Enter 提交、Escape 回退),不丢 emoji、tag、`[estimate::]`、`[actual::]`、`^blockrefs`、`🔁`、优先级符号。
- 作为用户,我希望**卡片操作键盘优先**:`1–4` 换象限、`←/→` 改天、`Space` 完成、`D` 日期弹窗、`Delete` 放弃、`E/Enter` 打开源文件、`Ctrl+Z` 撤销(20 深)。`/` 聚焦筛选框、`Ctrl+1–4` 切 tab、`Ctrl+T` 快速添加、`⌘/Ctrl+Shift+T` 打开看板。
- 作为用户,我想**一行快速添加任务**,支持自然语言日期(`今天 / tomorrow / 周六 / Mon`)。→ `Ctrl+T` QuickAdd 弹窗

## P2 · AI 协作者

- 作为 agent,我想用 **Obsidian 原生 CLI verb**,不用 shell 转义或 eval。→ `registerCliHandler`(Obsidian 1.12.2+)
- 作为 agent,我想**每行 list 输出的首列都是稳定 id**(`path:L42` 或 hash),这样 `grep/awk/cut` 直接能用。
- 作为 agent,我想要**幂等的写操作**,重跑不炸状态。→ 对已完成任务再跑 `done` 返回 `unchanged`,不报错
- 作为 agent,我想要**每次写都返回 `before/after` diff**,以便验证刚才发生了什么。
- 作为 agent,我想要 **`list` 和 `stats` 的 `format=json` 出口**,不必去刮人类友好输出。
- 作为 agent,我想要 **`stats days=N`** 暴露"估时准确率 + 各 tag 分钟数",让我能代替用户校准未来的估时。
- 作为 agent,我想要一套**跟人说话方式一致的时间筛选词**(`today / tomorrow / week / next-week / month / next-month / unscheduled / YYYY-MM-DD / FROM..TO`)。
- 作为 agent,我想要 **hash fallback 的 id 解析**,当行号失效也能找到任务;hash 歧义时应返回候选列表,不要猜。
- 作为 agent,我想要**增量时间操作**(`actual minutes=+30m`),这样记工时不必先读再写。
- 作为 agent,我想要**文档里明确推荐的工作流顺序**,不必反推 verb 的调用顺序。→ README "AI integration" 节

## P3 · 精力管理用户

- 作为用户,我想**给任务打象限 tag(`#1象限`–`#4象限`)**,未排期池按象限分组。
- 作为用户,我想**记估时(`[estimate:: 90m]`)和实际耗时(`[actual:: 75m]`)**。
- 作为用户,我想**在已完成 tab 看到近 7 天的准确率(`sum(actual)/sum(estimate)`)**,按周聚合,加 top-4 tag 分钟数——这样我能看清估时偏差在哪里、哪些 tag 吃掉我的时间。
- 作为用户,我希望**历史周默认折叠**,避免过去的记录把本周挤出屏幕。
- 作为用户,我希望 **`[-] ❌` 是一等"放弃"状态**,和"完成"区分开——这样我能回头复盘我放弃过什么,而不会把它当成完成计数。

## 横切关注点

- 作为用户,我想要**纯 markdown**——没有自定义格式、没有数据库。Obsidian Tasks、Dataview 和任何读纯文本的工具都保持可用。
- 作为用户,我希望 **zh/en 自动检测**(`localStorage.language`),不用手动配 locale。
- 作为用户,我希望**写操作是原子的**(`Vault.process`),mid-edit crash 不应损坏文件。
- 作为用户,我希望**读操作能快速跳过没有任务的文件**,在大 vault 上打开看板不卡。

## 空缺 / 待决策故事

以下是代码或 CHANGELOG 提过但 UX 故事还不明朗的点:

1. **循环任务(`🔁`)** — parser 保留了字段,但"完成一个循环任务 → 生成下一实例"这个故事不存在。用户的期望是什么?
2. **优先级符号(`🔺⏫🔼🔽⏬`)** — GUI 里保留但未用于排序/筛选。未排期和本周是否应按优先级排序?是否需要 `↑/↓` 快捷键改优先级?
3. **设置热更新** — CHANGELOG 标注为 known limitation。故事:"作为用户,我改了 `week starts on`,打开的看板应立刻反映,不用关了重开。"
4. **跨文件 nest 的撤销** — CHANGELOG 明说"只能用 git"。git 够用吗?还是需要 session 级撤销能追跨文件移动?
5. **子任务改期的表示** — 现在子任务拖到别的天是原地改 `⏳`,不是在目标日创建 `[[parent]] > child` wikilink。两种都是真实故事,但还没决定承诺哪个。
6. **多设备同步冲突** — Obsidian Sync / iCloud / git 三路抢同一行时,用户期望怎么表现?目前没有故事覆盖。
7. **提交到官方 Community Plugins** — 被 plugin id 里含 `obsidian` 子串挡住。故事:"作为用户,我希望能从社区插件浏览器里直接装 Task Center。" 需要改名才能解锁。
