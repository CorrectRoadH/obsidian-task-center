## 打开任务中心

4 种入口,都指向**同一个**看板 view:
1. 左侧边栏的 ribbon 图标(清单样式)。
2. 命令板(`Ctrl/Cmd+P` → "Open Task Board")。
3. 快捷键 `⌘/Ctrl+Shift+T`。
4. CLI `obsidian command id=obsidian-task-center:open`(给 agent 用)。

**首屏**:打开时默认显示**上次关掉时停留的 tab**;从没打开过就用设置里的 `defaultView`(默认"本周")。

## 任务卡长什么样

一张卡是一块矩形,包含四层信息:

```
┌──────────────────────────────────────────────────┐
│ ☐  去营业厅问携号转网          #2象限  📅 5-15   │  ← 头部:checkbox + 标题 + 象限 + deadline
│                                  est 90m         │  ← 时长条(估时 / 实际耗时)
│                                                  │
│   ├ ☐  写测试          est 30m                   │  ← 子任务(嵌在卡内)
│   └ ☐  跑 CI   ⏳ 4-27 est 60m                  │      └ 子的 ⏳ 和父不同 → 显 badge
│                                                  │
│                                 + 子任务  ✎     │  ← 底部:添加子任务 / 打开源文件
└──────────────────────────────────────────────────┘
```

- **头部**:`☐` / `✅` / `❌` + 标题 + tag 胶囊 + deadline 胶囊。
- **时长条**:估时和实际耗时都有就并排显示,差距大时 actual 变色(偏离 ±25% 算大)。
- **子任务区**:嵌在卡内,最多递归渲染到 3 层(再深也能展开,但默认折叠)。
- **底部**:`+ 子任务` 按钮、打开源文件按钮;hover 才出现。

**卡片的 tag**:`#1象限`–`#4象限` 用不同胶囊颜色(红 / 蓝 / 橙 / 灰);其他 `#tag` 用中性胶囊。

**选中态**:点一下卡变选中(有 outline),然后键盘快捷键作用于它。同一时刻只有一张卡被选中。

## 4 个 Tab

顶部 4 个 tab,每个右上角有个数字 badge = **该 tab 里 `[ ]` 待办的数量**;`[x]` 和 `[-]` 不计。

### 本周 / Week(`Ctrl+1`)

```
┌───────┬───────┬───────┬───────┬───────┬───────┬───────┐
│ 周一  │ 周二  │ 周三  │ 周四  │ 周五★ │ 周六  │ 周日  │
│ 4-20  │ 4-21  │ 4-22  │ 4-23  │ 4-24  │ 4-25  │ 4-26  │
├───────┼───────┼───────┼───────┼───────┼───────┼───────┤
│ [卡]  │ [卡]  │       │ [卡]  │ [卡]  │       │       │
│ [卡]  │       │       │       │ [卡]  │       │       │
│       │       │       │       │ [卡]  │       │       │
└───────┴───────┴───────┴───────┴───────┴───────┴───────┘
```

- 7 列 Mon–Sun(或 Sun–Sat,取决于设置)。今日列有高亮边框。
- 列内**按 deadline 升序**排;无 deadline 的在下,按创建时间倒序。
- 空列是可见的 drop zone(虚线外框)。

### 本月 / Month(`Ctrl+2`)

```
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│  28  │  29  │  30  │  4-1 │  2   │  3   │  4   │
│ [•]  │      │ [••] │      │      │ [•]  │      │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│  5   │  6   │  7   │ 8 ★  │ ...
```

- 日历网格。每格最多显示 3 张迷你卡(只有标题 + 小点),更多显示 "+N more"。
- 点格子展开成 popover 全列表。
- 每格都是 drop zone。

### 已完成 / Completed(`Ctrl+3`)

顶部固定**7 天统计**:

```
近 7 天准确率  actual/est = 0.83   within 11/18 (61%)
top tags       基建 240m  写作 180m  会议 120m  学习 75m
```

下面按周分组,**本周展开,过去的周默认折叠**,点一下展开。每条列 `[x]` 或 `[-]` 状态和完成日期。

### 未排期 / Unscheduled(`Ctrl+4`)

按**象限分组**(`#1象限` ~ `#4象限` 各一节,"其它"收尾),组内是 masonry 瀑布流。组头显示该组任务数。

## 未排期池 (bottom pool)

Week 和 Month 视图下方始终挂着一块固定高度的未排期池,masonry 瀑布流,**按 deadline 近的排前**。目的是"一眼看到池顶"。

---

## 垃圾桶

Week / Month 视图下方右侧,**一直可见**的桶形状 drop zone:

```
        ┌───────────┐
        │     🗑      │
        │  拖到这里  │
        │  = 放弃    │
        └───────────┘
```

不是删除。拖进去 = 任务变 `[-] ❌ 今日`,依然在 Obsidian 文件里,只是**从活动视图里消失**,能在"已完成" tab 的"放弃"部分回顾。

## 拖拽会发生什么

**核心**:同一个拖拽手势,松手在不同对象上语义不同。

| 松手在 ... | 发生什么 | 备注 |
|---|---|---|
| 本周某列 / 月视图某日期格 | 改 `⏳` 为那天 | US-121 / US-122 |
| 垃圾桶 | 标为放弃 `[-] ❌ 今日`,未打钩的子任务跟着放弃 | US-123 / US-124 |
| 另一张卡片的主体 | 这张卡变成那张的**子任务**,跨文件也行 | US-125 |
| 自己或自己的后代 | 拒绝,卡片抖一下回到原位 | US-126 |
| 视图外 / 非 drop zone | 卡片飞回原位 | |

拖的过程中:
- 源卡半透明跟手。
- 合法 drop zone **高亮**(亮色描边);非法 drop zone 变灰。
- 另一张卡作为 drop target 时,卡体边缘出现**虚线插入线**,提示"要嵌成它的子任务了"。

拖成子任务(US-125)跨文件时,完成一瞬间左下角浮起 toast:

```
已嵌入 FileB.md。这个动作不可撤销,如需回退请用 git。
```

(这个提示只在跨文件时出现;同文件内 nest 走正常撤销栈。)

## 父子任务的呈现

### 什么时候子作为独立顶层卡出现

| 父的状态 | 子的状态 | Week 里 |
|---|---|---|
| 父有 `⏳` 且终态非 done/dropped | 任意 | **父卡**显示,子嵌在父卡内;子**不再**单独出现 |
| 父无 `⏳` | 子有 `⏳` | 子作为独立顶层卡出现(因为父在 Week 不可见) |
| 父是 `[x]` / `[-]` / `#dropped` section 下 | 任意 | 父和所有后代都从 Week / Month / Unscheduled 消失 |

### 子任务的视觉层级

嵌在卡内时,每层用 `├` / `└` 做树枝,每层增加一级缩进:

```
☐ 父任务
  ├ ☐ 子任务 A
  │   └ ☐ 孙任务
  └ ☐ 子任务 B
```

### 子任务的 `⏳` badge 什么时候显示

- 和父**一样** → 不显示(默认就继承,显重复了没意义)
- 和父**不同** → 显示 `⏳ MM-DD` 小 badge(提醒"这个子任务要另一天做")

### 新建子任务时会发生什么

点卡底部 `+ 子任务`:
1. 卡底部长出一行单行输入框,自动聚焦。
2. 输入标题,Enter 提交,Esc 取消。
3. 新子任务继承父的 `⏳`。如果父和新子任务是**同一天创建**,新子任务不重复打 `➕ 今日`——避免满屏冗余时间戳。

### 放弃父任务时的级联

拖父到垃圾桶 → 父 `[-] ❌` →
- **未打钩**的子任务跟着 `[-] ❌`
- **已打钩** `[x]` 的子任务**保留不动**(它们是已完成的历史)
- 对孙、曾孙...递归同样规则

## 就地改名

点标题文字(不是 checkbox),标题变成一个 inline `<input>`:
- Enter 提交,失焦也提交。
- Esc 回退,恢复原文。
- **改名只改标题部分**:标题两侧的 tag / emoji / `[estimate::]` / `[actual::]` / `^blockref` / `🔁` / 优先级符号全部字节级保留。这是卡片上最容易引起用户"诶我的 emoji 呢"的地方,要严格守住。

## 键盘快捷键

选中一张卡后:

| 键 | 动作 |
|---|---|
| `1` / `2` / `3` / `4` | 切象限 tag(`#1象限`–`#4象限`);其他象限自动移除 |
| `←` / `→` | `⏳` 往前 / 往后一天 |
| `D` | 弹日期弹窗,手选 / 输入;`null` 清空 |
| `Space` | 切"完成"(`[x] ✅ 今日` ↔ `[ ]`) |
| `E` / `Enter` | 打开该任务所在 markdown 文件并定位到那行 |
| `Delete` / `Backspace` | 放弃(`[-] ❌ 今日`) |
| `⌘/Ctrl+Z` | 撤销 — 最近 20 步拖拽 / 改期 / 重命名 |

**不选中时**的全局快捷键:

| 键 | 动作 |
|---|---|
| `/` | 聚焦筛选输入框 |
| `Ctrl+1` / `2` / `3` / `4` | 切 tab(周 / 月 / 已完成 / 未排期) |
| `Ctrl+T` | 打开 QuickAdd 弹窗 |
| `⌘/Ctrl+Shift+T` | 打开 / 切到看板(全局) |

## 快速添加

`Ctrl+T` → 弹出单行输入框:

```
┌───────────────────────────────────────────────────────┐
│ 去营业厅问携号转网 #3象限 ⏳ 周六 [estimate:: 25m]    │
│                                                 ↵ 提交 │
└───────────────────────────────────────────────────────┘
```

- 支持 `#tag`、`⏳`、`📅`、`[estimate::]`、`[actual::]`。
- **自然语言日期**:`今天 / today / tomorrow / 明天 / 后天 / 周六 / Mon / Fri / YYYY-MM-DD`,都自动解析成 ISO。
- 没指定 `⏳` = 进未排期池。

**写到哪个文件**(优先级从高到低):
1. 如果在卡上按 `+ 子任务` → 父任务所在文件。
2. 今日的 daily note 如果存在 → 写进去。
3. 否则 → 设置里的 inbox(默认 `Tasks/Inbox.md`)。

## 动画

只在"让邻居别突然跳"的时候播放动画,其他时候都跳过。

| 动作 | 动画 | 时长 |
|---|---|---|
| 卡消失(拖到别的天 / 垃圾桶 / 完成 / 放弃 / `←/→` / `Delete` / `Space` / 日期弹窗) | 淡出 → 高度收缩 → 邻居上移 | ≤ 200ms |
| 卡出现(新加 / undo / 改回来) | 高度展开 → 淡入 | ≤ 200ms |
| 拖拽过程中 | 源卡跟手半透明,drop zone 边框动态高亮 | 实时 |
| **原地不动**的操作(例如:拖到当前列,按 `→` 但已是最后一天) | **不播放**,跳过 | — |

动画用 Web Animations API(原生 `element.animate`),走 GPU 层,不触发 layout。

## 状态栏挂件

Obsidian 底部状态栏(所有 view 都能看到)会挂一个小控件:

```
📋 5 today · ⚠ 2 overdue
```

- `5 today` = 今天 `⏳` 且 `[ ]` 的任务数。
- `2 overdue` = `📅` 已过期且 `[ ]` 的任务数。
- 点击 = 等价 `⌘/Ctrl+Shift+T` 打开 / 切到看板。

## 提示和错误

| 场景 | 呈现方式 |
|---|---|
| 写操作成功 | 无提示(让用户专注);卡片重渲染本身是反馈 |
| 拖到非法目标(如自己 / 后代) | 卡抖一下回到原位,底部 toast 2 秒 |
| 跨文件嵌套成功 | 底部 toast:"已嵌入 <file>。这个动作不可撤销,如需回退请用 git"(可 dismiss-and-remember) |
| 文件被外部改过 (`file_modified`) | toast:"这个文件刚被别处改了,已为你重新读取——再试一次"+ 自动刷新 |
| 撤销栈空 | `Ctrl+Z` 静默无反馈(避免噪音) |
| 快速添加的日期解析不出来 | 输入框下方 inline hint "没看懂日期:周陆(试试 周六 / today / 2026-05-01)" |
| 重命名提交时原标题已在磁盘被改 | 弹对话框:"这行刚被别处改动,你的改名是 `新标题`,磁盘上是 `外部标题`,要覆盖 / 放弃 / 合并?" |

**原则**:成功无噪音,失败可理解且可恢复。

## 设置页

Obsidian 设置 → 第三方插件 → Task Center:

| 项 | 默认 | 解释 |
|---|---|---|
| Default inbox path | `Tasks/Inbox.md` | 快速添加没命中 daily note 时写到这里 |
| Daily folder | `Daily` | 找今日 daily note 的文件夹 |
| Default view | Week | 第一次打开的 tab |
| Week starts on | Monday | 周一 / 周日第一天(ISO vs 美式) |
| Open board on startup | off | Vault 打开时自动开看板 |
| Stamp ➕ created date | on | 新任务自动打 `➕ 今日` |

> **注**:现在改设置要**关看板再开**才生效(US-904 在等决策)。

## 中英文

- 读 `localStorage.language`,**跟 Obsidian 界面语言走**。
- 缺 key 的翻译 **fallback 到英文原文**,不空也不报错。
- 界面里面向用户的词都走 i18n;**CLI 输出永远是英文**(方便 agent 解析,见 ARCHITECTURE)。
